What is DNS?

The Domain Name System (DNS) is often called the 'phonebook of the Internet'. It is a hierarchical, distributed database that translates human-readable domain names (like www.google.com) into numerical IP addresses (like 142.250.180.46) that computers use to identify each other.

Without DNS, you would need to memorise long strings of numbers to visit any website. DNS makes the Web human-friendly by allowing you to use memorable names instead.

DNS is invisible infrastructure — you never think about it, but every Internet action depends on it.

How DNS Resolution Works

  • You type www.example.com into your browser
  • Browser checks its local cache — if found, done
  • If not cached, your device queries a Recursive Resolver (usually your ISP's)
  • Resolver queries Root Name Servers → finds .com TLD servers
  • TLD servers point to example.com's authoritative nameserver
  • Authoritative server returns the IP address for www.example.com
  • Browser connects to that IP address and loads the website

DNS Record Types

DNS stores different types of records for different purposes.

  • A Record — maps a domain to an IPv4 address
  • AAAA Record — maps a domain to an IPv6 address
  • CNAME Record — creates an alias for another domain
  • MX Record — specifies mail servers for the domain
  • TXT Record — stores arbitrary text (e.g., for verification)

DNS Security

DNS was originally designed without security in mind. DNSSEC (DNS Security Extensions) was developed to add cryptographic authentication to DNS responses, protecting against attacks like DNS spoofing or cache poisoning where hackers redirect users to malicious sites.
